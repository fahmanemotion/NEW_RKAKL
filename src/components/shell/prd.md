Product Requirements & Technical Architecture Document: NEW_RKAKL1. Ringkasan ProdukNEW_RKAKL adalah aplikasi berbasis web berskala enterprise yang dirancang untuk mengelola siklus perencanaan, penganggaran, pelaporan, dan monitoring Rencana Kerja dan Anggaran Kementerian/Lembaga (RKAKL). Sistem ini memfasilitasi pembuatan usulan anggaran, manajemen Rincian Anggaran Biaya (RAB) berjenjang, proses peninjauan (review), pengawasan realisasi, hingga pencetakan dokumen legal.2. Teknologi & Desain Antarmuka (Tech Stack)Aplikasi ini dibangun dengan arsitektur modern untuk memastikan performa tinggi dan pengalaman pengguna yang optimal. Fokus pengembangan mencakup perancangan antarmuka (UI) yang lebih bagus, responsif, dan intuitif agar penyusunan data kompleks (seperti matriks anggaran) menjadi lebih mudah dipahami oleh pengguna operasional.Frontend & API: Next.js (App Router) dengan TypeScript.Desain Antarmuka (UI): Tailwind CSS dipadukan dengan komponen custom modern untuk menghasilkan visual dan interaksi yang rapi, bersih, dan profesional (khususnya pada Tree Grid dan dasbor).Backend & Database: Supabase (PostgreSQL) dengan fungsi Triggers, Functions, dan sistem migrasi terstruktur.Validasi Data: Zod (Type-safe validation di sisi klien dan server).3. Hak Akses Pengguna (User Roles)Peran (Role)Deskripsi & Hak Akses UtamaPembuat (Satker / Maker)Menginisiasi usulan anggaran, mengimpor/mengisi kertas kerja (RAB), mengedit volume dan harga satuan, serta mengajukan dokumen draf.Peninjau (Reviewer / Checker)Memeriksa kelayakan usulan yang diajukan Satker. Memiliki wewenang untuk menyetujui (Approve), menolak (Reject), atau mengembalikan dokumen untuk direvisi.Administrator (Admin Pusat)Mengelola Master Data (Referensi Satker, Kode Anggaran, Penandatangan), memantau aktivitas sistem, dan mengonfigurasi hak akses pengguna.4. Alur Kerja Utama (User Flow)A. Alur Pembuatan Usulan (Oleh Satker)Inisiasi: Login -> Navigasi ke Dasbor -> Klik "Buat Usulan Baru".Pengisian Data: \* Opsi 1: Impor template Excel standar (sistem memvalidasi anti-duplikat).Opsi 2: Input manual membangun hierarki (Program -> Kegiatan -> KRO -> Detail Jenis Belanja).Kalkulasi: Mengisi form Detail Volume (volume, satuan, harga). Sistem menghitung total secara real-time ke semua struktur atasnya.Pengajuan: Meninjau rekapitulasi -> Klik "Ajukan Usulan" (Status berubah menjadi Pending Review).B. Alur Peninjauan (Oleh Reviewer)Pemeriksaan: Menerima notifikasi -> Membuka daftar usulan Pending Review -> Mengevaluasi Tree Grid RAB.Keputusan: Memberikan persetujuan (Approved) atau catatan revisi (Return to Draft).5. Arsitektur Keamanan Database (Row Level Security)Perlindungan data dilakukan di tingkat database (PostgreSQL) menggunakan fitur Row Level Security (RLS) dan injeksi otorisasi JWT.Isolasi Satker (Zero Trust): Kebijakan RLS memastikan bahwa akun tingkat Satker hanya dapat melakukan SELECT, INSERT, UPDATE pada data yang memiliki ID Satker mereka sendiri. Satker A secara sistem tidak akan bisa melihat draf Satker B meskipun mencoba menembus API.Akses Reviewer: Diberikan hak akses lintas Satker, tetapi hanya untuk dokumen yang statusnya sudah diajukan (bukan draf internal).Proteksi Master Data: Data referensi (KRO, Standar Biaya, dll) terbuka untuk dibaca oleh semua pengguna, tetapi hak mengubah dikunci mutlak hanya untuk Administrator.6. Rancangan Integrasi (API Endpoints)EndpointMethodFungsi Utama/api/v1/anggaran/kertas-kerjaGETMenarik data hierarki (Tree View) dari dokumen RAB. Mendukung lazy loading./api/v1/anggaran/detail-volumePOSTMenyimpan perubahan rincian belanja dan memicu kalkulasi di database./api/v1/anggaran/copyPOSTMenyalin struktur anggaran utuh dari dokumen lama ke usulan periode baru./api/v1/anggaran/import-excelPOSTMengekstrak dan memvalidasi baris dari file kertas kerja Excel untuk disimpan.7. Skema Validasi Integritas DataSistem menggunakan Zod untuk memastikan tidak ada anomali data (misal: nilai minus atau tipe data salah) yang masuk ke database. Semua volume dan harga satuan dipaksa harus bernilai positif (> 0), dan format teks tervalidasi secara ketat sebelum diproses oleh backend.8. Mekanisme Ekspor & PelaporanA. Ekspor Kertas Kerja Terbuka (Excel)Metode: Server-side generation menggunakan pustaka exceljs.Proses: Sistem menarik data hierarki (Tree), melakukan perataan kolom (flattening), dan menyuntikkan gaya secara dinamis (teks tebal dan warna latar untuk level atas, serta indentasi untuk sub-level).B. Pencetakan Dokumen Resmi (PDF / Cetak RAB)Metode Ringkas: Halaman khusus pelaporan menggunakan utilitas CSS @media print untuk menampilkan kop surat resmi, tabel baku, dan memecah halaman secara otomatis saat pengguna mencetak via peramban web.Metode Baku: Generasi PDF statis dari backend untuk memastikan format tabel dan area tanda tangan basah terkunci sempurna tanpa terpengaruh jenis perangkat pengguna.
